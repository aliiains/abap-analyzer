Ты – высококвалифицированный Python-разработчик с глубокими знаниями принципов SOLID, модульной архитектуры и тестирования. Твоя задача – написать десктопное приложение на Python (с использованием Tkinter), которое является анализатором кода на языке ABAP (с минимальной поддержкой синтаксиса, но способное работать с любым текстом). Приложение должно иметь графический интерфейс: текстовое поле для ввода ABAP-кода, кнопка "Анализировать", область для отображения результатов (метрики, выводы) и кнопка "Скачать результат" (сохраняет в текстовый файл).

Функциональные требования:

Пользователь вставляет код ABAP в окно ввода.
По нажатию "Анализировать" приложение выполняет:
Подсчёт метрик: количество строк кода (LOC), количество строк комментариев, количество пустых строк, количество операторов (например, ключевых слов ABAP: DATA, IF, LOOP, CALL FUNCTION и т.д.), глубина вложенности (максимальная), количество функций/процедур (FORM), количество "магических чисел".
Генерацию выводов/рекомендаций на основе метрик: например, если вложенность >5 – "Рекомендуется рефакторинг", если длинная строка >120 символов – "Превышение лимита длины строки", если мало комментариев – "Недостаточно документирования" и т.п.
Результат отображается в правой панели (или под полем ввода) в читаемом виде (например, в виде списка или таблицы).
Кнопка "Скачать результат" сохраняет отчёт в текстовый файл (включает все метрики и выводы).
Архитектурные требования (SOLID + модульность):

Раздели код на модули (пакеты):
gui.py – весь код графического интерфейса (Tkinter). Класс App, который инициализирует окно и связывает виджеты.
parser.py – парсер ABAP (анализ строк, подсчёт LOC, комментариев, пустых строк, операторов, вложенности и т.п.). Класс AbapParser с методами.
metrics.py – вычисление метрик на основе данных от парсера (например, класс MetricsCalculator).
report.py – формирование отчёта (текстового и для отображения в GUI).
tests/ – папка с unit-тестами для каждого модуля (минимум 3-4 теста на модуль).
main.py – точка входа, создание экземпляра App и запуск.
Следуй принципам SOLID: каждый класс имеет одну ответственность, зависимости инвертированы (используй абстрактные базовые классы или протоколы), открытость для расширения (например, добавление новых метрик через подклассы или стратегии).
Код должен быть читаемым: используй type hints, docstrings, понятные имена переменных, соблюдай PEP8.
Технические детали:

Используй стандартную библиотеку Python (tkinter, abc, typing). Для тестов – pytest (не unittest). Все тесты должны проходить.
Общий объём кода (все файлы вместе, включая тесты) должен быть не менее 2000 строк. Если в твоём решении будет меньше – добавь больше метрик, тестов, комментариев или дополнительных фич (например, визуализация сложности, подсветка синтаксиса в поле ввода).
Код ABAP для примера можешь использовать минимальный (несколько строк), но анализатор должен корректно обрабатывать вложенные структуры и ключевые слова.
Файлы должны быть готовы к запуску: pip install pytest, затем pytest в корне проекта; python main.py – открывает окно.
Вывод:
Напиши полный код каждого файла с комментариями. Выводи весь код в виде блоков с указанием имени файла (например, # gui.py). Убедись, что суммарное количество строк не менее 2000.

Важно: Не сокращай код, не используй ..., пиши полностью. Все функции и классы должны быть реализованы. После завершения напиши краткую инструкцию по запуску и тестированию.
