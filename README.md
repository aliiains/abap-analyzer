# ABAP Code Analyser

Настольное приложение для статического анализа кода на языке ABAP (SAP).
Написано на Python с графическим интерфейсом Tkinter. Не требует интернета,
работает полностью локально.

## Что умеет приложение

- Анализирует исходный код ABAP и выявляет потенциальные проблемы качества
- Вычисляет метрики: цикломатическую сложность, глубину вложенности, длину строк,
  количество строк кода, комментариев, пустых строк и многое другое
- Подсвечивает синтаксис ключевых слов, строк, чисел и комментариев
- Визуализирует глубину вложенности цветовой тепловой картой (от янтарного до бордового)
- Выводит структурированный отчёт с замечаниями трёх уровней:
  `ERROR` (критично), `WARNING` (предупреждение), `INFO` (информация)
- Сохраняет отчёт в текстовый файл
- Загружает `.abap` и `.txt` файлы с диска
- Поддерживает вставку кода из буфера обмена (Ctrl+V / Command+V)

## Описание кнопок

### Панель инструментов

#### ▶ Анализировать
Запускает полный анализ кода, введённого в левую панель.

- Парсит исходный код (разбивает на токены, строит структуру)
- Вычисляет все метрики качества
- Формирует отчёт с замечаниями
- Обновляет правую панель: вкладку «Отчёт» и вкладку «Метрики»
- Если включена тепловая карта — перекрашивает строки по глубине вложенности
- Если поле ввода пустое — показывает предупреждение
- Сочетание клавиш: **Ctrl+Enter**

---

#### 📂 Открыть файл
Открывает диалог выбора файла с диска.

- Поддерживаемые форматы: `.abap`, `.txt`, любой текстовый файл
- Содержимое файла подставляется в редактор кода
- Автоматически применяется синтаксическая подсветка
- После загрузки в статусной строке отображается имя файла

---

#### 💾 Скачать результат
Сохраняет текст отчёта в файл на диске.

- Активируется только после успешного анализа (до анализа кнопка неактивна)
- Открывает диалог «Сохранить как» с предложенным именем `abap_report.txt`
- Сохраняет полный текстовый отчёт: метрики и все замечания

---

#### 🎨 Подсветка: ВКЛ / ВЫКЛ
Переключает синтаксическую подсветку кода в редакторе.

- **ВКЛ** — ключевые слова ABAP выделены синим и жирным; строки — зелёным;
  числа — жёлтым; комментарии — серым; строки длиннее 120 символов — тёмно-красным фоном
- **ВЫКЛ** — весь код отображается одним цветом (обычный текст)
- При переключении статусная строка показывает текущее состояние
- Подсветка включена по умолчанию при запуске

---

#### 🌡 Тепловая карта: ВКЛ / ВЫКЛ
Переключает визуализацию глубины вложенности блоков кода.

- **ВКЛ** — каждая строка получает цвет фона в зависимости от уровня вложенности:
  - 0 уровней — стандартный тёмный фон
  - 1 уровень — едва тёплый оттенок
  - 2–4 уровня — оранжево-янтарный градиент
  - 5–7+ уровней — насыщенный красно-бордовый
- **ВЫКЛ** — все строки возвращаются к стандартному фону
- Подсказка по цветам всегда видна в легенде под редактором
- Тепловая карта включена по умолчанию; обновляется после каждого анализа

---

### Редактор кода (левая панель)

Текстовое поле, в которое вводится или вставляется ABAP-код.

- Поддерживает прокрутку по горизонтали и вертикали
- Вставка из буфера обмена: **Ctrl+V** (Windows/Linux) или **Command+V** (macOS)
- Счётчик строк в правом нижнем углу обновляется при каждом нажатии клавиши
- Подсветка применяется в реальном времени при вводе

---

### Панель результатов (правая панель)

Содержит две вкладки:

#### Вкладка «Отчёт»
Полный текстовый отчёт после анализа:
- Сводные метрики (строки кода, сложность, вложенность и т.д.)
- Список замечаний с указанием номера строки, уровня и описания

#### Вкладка «Метрики»
Таблица со всеми вычисленными числовыми метриками:
- Значение каждой метрики
- Оценка: ✅ норма / ⚠️ предупреждение / ❌ критично

## Метрики

| Метрика | Описание |
|---------|----------|
| Всего строк | Общее количество строк в файле |
| Строк кода | Строки, содержащие исполняемые инструкции |
| Строк комментариев | Строки, начинающиеся с `*` или `"` |
| Пустых строк | Строки без содержимого |
| Цикломатическая сложность | Количество ветвлений (IF, CASE, LOOP и т.д.) |
| Макс. глубина вложенности | Максимальный уровень вложенных блоков |
| Длинных строк (>80 символов) | Строки, превышающие рекомендуемую длину |
| Очень длинных строк (>120) | Строки, превышающие допустимую длину |
| Методов / функций | Количество обнаруженных подпрограмм |
| Средняя длина строки | Средний размер строки кода в символах |
